## 二、病人管理

### 新增病人

1. 點左側選單「病人管理」
2. 右上角點「新增病人」
3. 填入基本資料（姓名、病歷號、出生日期、癌別）
4. 點「儲存」

> **注意：** 病歷號一旦建立後不能修改，請確認輸入正確。

### 查看病人詳情

點擊病人卡片即可進入詳情頁，共有 7 個分頁：

| 分頁 | 內容 |
|------|------|
| 總覽 | 快速查看病人狀態與近期事件 |
| 基本資料 | 個人資料、聯絡方式 |
| 分期 | 臨床分期與病理分期 |
| 收案/癌別 | 癌別特定資料（依癌別不同） |
| 事件記錄 | 所有治療事件時間軸 |
| 指標 | 該病人的品質指標達成狀況 |
| 追蹤 | 後續追蹤計畫 |

### 多重癌別病人

同一病人有兩種以上癌別時，系統會在姓名旁顯示「多重癌 ×2」紅色標記，並在詳情頁提供其他癌別的跳轉連結。

### 篩選病人

病人清單頂部可依以下條件篩選：
- 癌別（全部 / 指定癌別）
- 狀態（在治 / 追蹤 / 結案 / 已歿）
- 個管師（僅 admin 可看到所有人）

## 三、事件記錄

### 新增事件

在病人詳情頁「事件記錄」分頁，點右上角「新增事件」。

常見事件類型：

| 類型 | 說明 |
|------|------|
| 手術 | 術式、切緣、淋巴結 |
| 化療 | 藥物、週期數、是否新輔助 |
| 放療 | 部位、劑量（Gy）、分次數 |
| 標靶治療 | 藥物名稱 |
| 荷爾蒙治療 | 藥物名稱 |
| MDT | 多專科會議討論記錄 |
| 追蹤回診 | 一般門診追蹤 |

### requiresMDT 標記

新增事件時，若該事件需要在 MDT 討論，勾選「需要 MDT 討論」。  
系統會自動將該病人列入下次 MDT 的「必要提報」名單。

## 四、工作中心

工作中心彙整所有需要關注的病人，分三個優先層級：

### 緊急（紅色）
- 待辦事件已逾期

### 本週待辦（黃色）
- 事件到期日在本週
- 術後追蹤時間點（第28天、第88天…）
- 放療或化療進行中
- 距上次訪視超過 75 天

### 需關注（灰色）
- 新收案超過 30 天尚未有治療事件
- 失聯超過 12 個月

### 操作

點擊工作中心的任何一筆，會直接跳到該病人的詳情頁。

### 建立 MDT 會議

1. 點左側「多專科會議」→「行事曆」分頁
2. 行事曆上有**虛線徽章**（建議排程）→ 點擊即進入建立畫面
   - 或點右上角「新建會議」手動建立
3. 選擇癌別群組，系統自動帶入：
   - 建議日期（可修改）
   - 開會時間（可修改）
   - 地點（可修改）
   - 負責個管師
4. 點「建立會議」

### 複製上次會議

在已有的會議詳情面板，點「複製」按鈕：
- 選擇新的日期
- 系統複製病人名單，但清空結論和追蹤計畫
- 適合每月例行會議使用

---

### 進入會議室

點擊行事曆上的**實色徽章**（已建立的會議）→ 展開詳情 → 點「進入會議 →」

#### 五個討論區塊

| 區塊 | 說明 | 名單來源 |
|------|------|---------|
| ⭐ 新個案討論 | 上次 MDT 後新收的病人 | 系統自動列出，勾選確認 |
| 📈 前期追蹤 | 上次有結論、這次回報進度 | 系統自動列出 |
| ⚠️ 必要提報 | 符合國健署基準2.4的病人 | 系統自動偵測（⚡標記） |
| 📋 醫療小組 | 個管師回覆追蹤 | 有醫療小組事件卡的病人 |
| 👤 特殊議程 | 臨時加入的病人 | 手動搜尋加入 |

#### 填寫討論內容

點擊病人卡片展開填寫欄位：
- **影像結果**：CT/MRI/PET 所見
- **病理結果**：確診資訊
- **討論要點**：開會討論的重點
- **結論與建議**：有結論時邊框轉綠色
- **後續追蹤計畫** + 追蹤日期

#### 手動加入病人

點每個區塊右上角「加入病人」→ 輸入姓名或病歷號搜尋。

---

### 會後填寫

開完會後，點 Header 的「**會後填寫**」按鈕：
- 一次看所有討論個案
- 可補填或修改討論摘要、結論、後續追蹤
- **帶入下次追蹤** toggle：  
  - 開（預設）→ 下次 MDT 自動列入前期追蹤名單  
  - 關 → 下次 MDT 不再自動出現

---

### 完成會議

點「**完成會議**」：
- 會議狀態改為「已完成」
- 有結論的病人自動新增 MDT 事件卡
- 有 `requiresMDT` 的事件卡標記為已討論

| 按鈕 | 說明 |

## 六、指標管理

顯示國健署 115 年強制申報的 60 項品質指標達成狀況。

### 待補項目

列出每位病人缺少的指標資料，點擊病人名稱直接跳到對應的詳情頁填寫。

### 指標總覽

依癌別分組顯示每項指標的達成率。

### 達成報告

各癌別的整體達成情況摘要。

### 趨勢預警

距離申報期限的倒數提醒，以及需要特別關注的指標。

### 資料匯出

匯出指標資料供國健署申報使用。

## 八、系統工具

### 重置資料庫

清除所有資料，重新載入預設主檔和測試病人。  
**！ 此操作無法復原，請謹慎使用。**

操作步驟：
1. 點左側「系統工具」
2. 點「重置資料庫」
3. 輸入 4 位確認碼
4. 點「確認重置」
5. 等待頁面自動重新載入

### 科別/醫師同步

當系統更新新增了科別或醫師時，點此按鈕補入最新資料（不刪除現有資料）。

### 刪除測試病人

刪除所有病歷號以 `T-` 開頭的測試病人及其相關事件和 MDT 記錄。

### 備份與還原

- 點「匯出備份」下載 JSON 格式的備份檔
- 點「匯入備份」選擇備份檔還原

## 常見問題

**Q：資料儲存在哪裡？**  
A：儲存在這台電腦的 Chrome/Edge 瀏覽器中。換電腦或換瀏覽器資料不會自動同步，請定期備份。

**Q：系統打不開，一直轉圈？**  
A：在地址列輸入網址後，按 `Ctrl+Shift+R`（強制重新整理），通常可以解決。

**Q：MDT 會議建立後日期要改？**  
A：目前不支援修改已建立的會議日期，可以刪除後重新建立，或使用「複製」功能建立新日期的會議。

**Q：病人的多重癌怎麼處理？**  
A：建立兩筆獨立的病人記錄，病歷號相同、癌別不同。系統會自動偵測並在兩筆記錄上顯示「多重癌」標記。
